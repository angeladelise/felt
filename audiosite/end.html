Importance of Inclusive Design

39 Million

people in the world are blind. Individuals that are blind rely on screen readers to navigate through digital spaces. If a website is not accessible with this assistive technologies, it hinders the ability of this group to access information.

244 Lawsuits

have been filed from January 2015 to October 2016 for not complying with web accessibility standards; 148 of them have been in the retail space.

Accessible Web Designs

WebAim is an accessible web design. The links and images contain proper tagging to provide seamless interaction with screen readers.

This site also has a logical hierarchy of headers which eases the navigation system. All of the forms contain labels so users are able to identify the proper content in each section.


CouchSurfing.org is a website that connects hosts with travelers to aid in their experience of a location.

This site also has a logical hierarchy of headers which eases the navigation system. The buttons contain proper labeling so users are able to identify the contents of each button.


Inaccessible Web Designs

Amazon is the largest online store; providing a tremendous amount of products to their users. An image or video without an "alt" tag will make the screen reader read the file name of the item. Non-descriptive file labeling leaves the user completely unaware about the content of the item.

For example, when on www.Amazon.com the screen reader states, “2593218011._ CB308835085_ SR120,50_ .jpg” when it is on the Hanes logo. This long slew of letters and numbers contain no information about the image. This coding flaw prevents screen reader users from understanding this information.


Ebay contains a grid of images and links to access information about their selection of products. Using JavaScript on a site requries ARIA tags to make information accessible.

This example shows what occurs when a screen reader selects an item from the 'Holiday Gatherings' section. A pop up div appears; however, the screen reader cannot access any of the content. The reader does not indicate that a pop up is on the screen and does not allow the user to interact with any of the information. The content is completely inaccessible to the user.


Section 508 Guidelines

Section 508 of the accessibility guidelines provides information about tagging for screen reader interaction.

Every non-text element needs to contain a text alternative (e.g., via "alt", "longdesc", or in element content).

Electronic forms must be accessible with assistive technologies. Users must be able to access information, input field data and submit the form.

All multimedia production, including video must include an audio description.

A variety of color and contrast settings should be provided for the user to control.
